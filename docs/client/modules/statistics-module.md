# 客户端统计模块设计

## 功能范围

Android 和 Web 都提供统计视图：

- 金额比例饼图。
- 时间趋势折线图。

客户端只负责渲染和交互，聚合数据来自服务端统计 API。

## 数据来源

- `GET /api/stats/category`
- `GET /api/stats/timeline`

请求参数：

- 时间范围。
- 成员。
- 银行名称。
- 一级分类或二级分类。
- 比较属性：一级分类、二级分类、使用人、银行。
- 方向：收入、支出、转账。
- 折线粒度：日、周、月。

统计页首次进入时默认查询最近一个月：结束日期为今天，起始日期为一个月前。用户可以手动清空单个日期字段以查询开放时间范围；“清除过滤”恢复到该默认日期范围。

## 金额展示

服务端返回 `amount_cent`。客户端展示时统一除以 100：

- `1234` -> `12.34`
- 单位显示为 RMB 或 `¥`。

## 饼图

饼图展示当前比较属性的占比：

- 默认按一级分类聚合。
- 选择按一级分类比较时，不再显示一级分类和二级分类过滤。
- 选择按二级分类比较时，只允许用一级分类、使用人、银行和日期范围过滤。
- 选择按使用人比较时，不再显示使用人过滤。
- 选择按银行比较时，不再显示银行过滤。
- 只统计 `deleted_at IS NULL` 的交易。

## 折线图

折线图展示时间趋势：

- 支持按日、按周、按月。
- 横轴为时间桶。
- 纵轴为聚合金额。
- 每个比较属性值绘制一条不同颜色折线，并和饼图/明细行颜色保持一致。
- 折线图默认不显示点位金额；用户点击下方图例色点或名称后，只显示同色折线的所有点位金额，再次点击隐藏。
- 页面过滤器可按统计方向、允许的分类、使用人、银行和日期范围刷新折线图。

## Flutter 复用

统计页面和图表组件应尽量在 Android 和 Web 复用。若某个图表库在 Web 或 Android 表现不一致，必须通过组件封装保持页面接口一致。

首版图表实现：

- 使用 Flutter `CustomPainter` 或轻量自绘 widget 绘制饼图和折线图。
- 不默认引入第三方图表库。
- 图表只在统计页加载数据并绘制。
- 数据点过多时由服务端按 day/month 聚合，客户端不渲染过密点。

## 验收标准

- Android 和 Web 统计页面 UI 一致。
- 客户端不自行扫描全量交易做主统计。
- 金额展示无浮点持久化依赖。
- 图表实现不引入大型依赖，构建体积变化需要记录。
